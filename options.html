Propuesta comercial de blog

Objetivo

Tres formas de añadir un blog a Skin by Karla

La prioridad es que Karla pueda publicar contenido sin tocar código: título, texto e imagen destacada. Estas son las tres vías posibles, ordenadas por facilidad, control y esfuerzo técnico.

Decisión clave

Elegir si el blog debe vivir fuera de la web para empezar rápido, o dentro de la web para ganar control y posicionamiento a largo plazo.

Más rápido

Substack: blog y newsletter en una sola herramienta.

Más completo

WordPress: control total, pero con más coste y mantenimiento.

Mejor equilibrio

GitHub Pages + CMS: blog integrado sin migrar toda la web.

Opciones

Qué puede elegir Karla

1

Web actual + Substack

Mantenemos la web actual en GitHub Pages y añadimos un enlace a Substack. Cada publicación funciona como artículo y también como newsletter enviada por email.

Ideal para empezar rápido y validar el contenido.
  • Muy fácil para Karla.
  • Resuelve blog y newsletter a la vez.
  • Menor coste inicial.
  • Menos control visual y SEO repartido entre web y Substack.

2

Migrar todo a WordPress

Portamos la web completa a WordPress para que Karla tenga un panel tradicional donde crear posts, subir imágenes, editar páginas y controlar categorías.

Ideal si el blog será una pieza central del negocio.
  • Control total del blog y del contenido.
  • Panel conocido y muy completo.
  • Permite crecer hacia más funcionalidades.
  • Requiere hosting, mantenimiento, seguridad y más horas.

3

GitHub Pages + CMS

Mantenemos la web actual y añadimos un blog estático con Jekyll. Karla podría publicar desde Decap CMS o desde un flujo asistido con ChatGPT/Codex.

Ideal para integrar el blog sin pagar hosting extra.
  • El blog vive dentro de la web principal.
  • Buen equilibrio entre coste, SEO y control.
  • No requiere migrar toda la web.
  • Necesita configuración técnica inicial y buen flujo de publicación.

Recomendación

Empezar simple y escalar si el contenido funciona

La vía más prudente es empezar con Substack si Karla quiere validar la constancia del blog y construir una lista de emails sin fricción. Si el contenido empieza a ser importante para su marca, el siguiente paso natural sería integrar el blog en la web con GitHub Pages, Jekyll y un CMS.

WordPress tiene sentido si desde el primer momento el blog va a ser una parte central del negocio y se acepta una inversión mayor en configuración, hosting y mantenimiento.

Mensaje comercial sugerido

Podemos empezar con una solución rápida y sencilla, y si el blog crece, lo integramos dentro de la web principal sin perder el trabajo realizado.
